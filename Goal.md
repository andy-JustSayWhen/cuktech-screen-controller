# AP01 固件安全规则整理目标

## 当前目标

1. 将固件制作、修改、安装、回滚和恢复的安全规则集中维护在 `agent-docs/firmware-safety.md`。
2. 在根目录 `AGENTS.md` 中保留项目权威文件顺序，并声明固件操作前必须读取安全规则。
3. 完善忽略规则，避免本机环境、凭据、索引、日志、构建文件和固件产物进入版本库。

## 范围

- 本次只整理项目规则和文档，不构建、不上传、不安装任何固件。
- 保留 `Depends.md`、`.env` 和现有设备运行状态。
- 不修改画面生成、局域网服务、固件构建或设备控制代码。

## 执行顺序

1. 检查本地改动及项目内引用。
2. 修复失效引用和权威文件缺失。
3. 检查文档差异、忽略规则和敏感文件状态。
4. 提交并推送到当前远端分支。

## 验收标准

- `Goal.md`、`Depends.md` 和 `.env` 的职责保持清晰。
- 固件安全规则能够从 `AGENTS.md` 直接定位。
- 项目说明中不存在因本次整理产生的失效文件链接。
- `.env` 继续保持仅当前用户可读写且不进入提交。
- Git 差异检查通过，提交成功推送到当前远端分支。

## 安全停止条件

- 发现凭据、设备标识、固件文件或临时下载地址进入待提交内容时停止推送。
- 发现本次改动涉及设备操作或程序行为时停止并重新确认范围。
